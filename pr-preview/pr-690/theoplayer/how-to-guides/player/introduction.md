# Introduction

The Player API is the top-level API. This API exposes:

* basic functions to manipulate video playback, such as start and pause;
* properties to check states and values, such as the playhead position;
* events to track changes.

The list below overviews how-to articles related to the Player API.

* [How to measure time-to-first-frame](/documentation/pr-preview/pr-690/theoplayer/how-to-guides/player/how-to-measure-time-to-first-frame.md)
* [How can we track the first play(ing) event?](/documentation/pr-preview/pr-690/theoplayer/how-to-guides/player/how-can-we-track-the-first-playing-event.md)

## Events[​](#events "Direct link to Events")

As a developer, you can programmatically attach event listeners to different APIs. Event listeners are often used to [implement a custom UI](/documentation/pr-preview/pr-690/theoplayer/how-to-guides/ui/how-to-build-chromeless-ui.md) or to [implement a custom analytics integration](/documentation/pr-preview/pr-690/theoplayer/how-to-guides/analytics/custom-analytics-integration.md).

The THEOplayer API exposes many events across different interfaces, as can be seen in the graphic below. Furthermore, most events exist in all SDKs (Web, iOS-based, Android-based and Roku).

![THEOplayer API Interfaces and Events](/documentation/pr-preview/pr-690/assets/images/analytics-events-185d6ab52b556af08d8300b545ae5876.png "THEOplayer API Interfaces and Events")
